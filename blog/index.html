Webpack 初上手

前端世界风起云涌，现在最火的应该就是React，而在14年instagram开发者介绍了构建React项目的前端工程化工具——Webpack。可以参看 webpack-howto

而我对于Webpack的入门是看了另一份文档，因为有中文版。参看 react-webpack-cookbook 。

一天时间看下来，Webpack到底干了些什么事情呢？

首先，Webpack实现了前端模块化编程，可以用写nodejs的方式来写前端代码，可以requrie进来nodejs浩如烟海的现成lib，这几乎改变了前端编程的方式。

ES6 模块

import MyModule from './MyModule.js';

CommonJS

var MyModule = require('./MyModule.js');

AMD

define(['./MyModule.js'], function (MyModule) {

});

其次的作用就是项目打包的功能，像Grunt，Gulp所做的事情，而Webpack是通过loader来实现的。

列举几个必用的loader：

  • css-loader!style-loader
    这是加载css代码的loader，npm install css-loader style-loader —save-dev。如果是用less或sass来写css，还有less-loader和sass-loader可以用，安装方法也是一样的。

  • url-loader
    npm install url-loader —save-dev来安装加载器，它会把需要转换的路径变成 BASE64 字符串，在其他的 Webpack 书中提到的这方面会把你 CSS 中的 “url()” 像其他 require 或者 import 来处理。意味着如果我们可以通过它来处理我们的图片文件。

  • babel-loader
    这是我最喜欢，也算是最重要的loader了，有了它可以转换react的jsx文件，但更重要的是，有了它就可以用ES6来写React代码了，真的是很爽。

另外，还有一些Webpack的特色功能，比如多重入口，可以通过下面代码感受一下：

webpack.config.js
var path = require('path');
var webpack = require('webpack');
var node_modules_dir = path.resolve(__dirname, 'node_modules');

var config = {
  entry: {
    app: path.resolve(__dirname, 'app/main.js'),
    mobile: path.resolve(__dirname, 'app/mobile.js'),
    vendors: ['react'] // 其他库
  },
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].js' // 注意我们使用了变量
  },
  module: {
    loaders: [{
      test: /\.js$/,
      exclude: [node_modules_dir],
      loader: 'babel'
    }]
  },
  plugins: [
    new webpack.optimize.CommonsChunkPlugin('vendors', 'vendors.js')
  ]
};

module.exports = config;

还有就是懒加载入口的功能，比如

main.js (使用 ES6 语法)
import React from 'react';
import Feed from './Feed.js';

class App extends React.Component {
  constructor() {
    this.state = { currentComponent: Feed };
  }
  openProfile() {
    require.ensure([], () => {
      var Profile = require('./Profile.js');
      this.setState({
        currentComponent: Profile
      });
    });
  }
  render() {
   return (
      return <div>{this.state.currentComponent()}</div>
    );
  }
}
React.render(<App/>, document.body);

以上是我看了一天Webpack的粗浅体会，肯定还是很肤浅很初级，但重点在于记录这个开始，以后要多多用Webpack+react+ES6写一些应用，应该会很好用。

最后是我写了一个webpack+react的练手之作，献丑GitHub。

去啊面试题|试答

在拔赤大神的github页面看到阿里航旅事业部的面试题，浏览一遍，有的问题有一些明白，有一些问题则不太懂，索性转到这里，然后不管自己理解还是查阅资料，将各个问题自己给出的答案贴出来，一方面记录自己的学习，一方面如有大神指教就更更好了。

1. CSS 盒子模型，绝对定位和相对定位

(1) 清除浮动，什么时候需要清除浮动，清除浮动都有哪些方法?

答：父元素在包裹一个float的子元素，如果父元素的height为auto时，父元素无法自动伸长以适应内容的高度，这样内容就会溢出，影响布局，这时就要清除浮动。
清除浮动的方法：

  1. 给父级元素添加height，在height可以计算的情况下可以用。
  2. 在子元素后面添加类名为clear的元素,给以css代码.clear{clear:both}。这种方法的缺点是添加了无意义的html标签。
  3. 给父级元素添加 overflow:hidden 。
  4. 利用:after伪元素，给父级一个clearfix的类，然后clearfix:after{content: “020”; display: block; height: 0; clear: both; visibility: hidden;}

第四种应该是用得最多，最完美的清除浮动的方法。

(2) 如何保持浮层水平垂直居中

答：其实不太理解这个问题的确切意思，是在考绝对定位吗？

通过使用百分比绝对定位,与外补丁负值的方法，负值的大小为其自身宽度高度的一半”达到让层垂直居中的效果。
div.innerbox {
position:absolute;
top:50%;
left:50%;
margin:-100px 0 0 -100px;
width:200px;
height:200px;
}

(3) position 和 display 的取值和各自的意思和用法

答：position有5个值：static，relative，absolute，fixed，inherit。

  1. static: 这是不设置position值的默认值。在一般情况下不需要显式设置。在默认static下，top,bottom,left,right及z-index属性都是被忽略的。
  2. relative: 相对定位，顾名思义，relative就是相对元素在文档流中原本的位置，通过TBLR进行移动。实际上元素在文档流中依然占据原来的位置，只是视觉上移动了。
  3. absolute: 绝对定位。脱离文档流，生成绝对定位的元素。如果元素的父级元素中有relative的元素，那么就相对此元素进行TBLR移动，如果父级中没有relative的元素，就相对body元素进行TBLR移动。
  4. fixed：固定定位。脱离文档流，相对于当前浏览器窗口进行TBLR移动。
  5. inherit：继承父级的定位。
    display值有：none，block，inline，inline-block等。

(4) 样式的层级关系，选择器优先级，样式冲突，以及抽离样式模块怎么写，说出思路，有无实践经验

(1) JavaScript 里有哪些数据类型，解释清楚 null 和 undefined，解释清楚原始数据类型和引用数据类型。比如讲一下 1 和 Number(1) 的区别

3. JavaScript 的面向对象

(1) JS 模块包装格式都用过哪些，CommonJS、AMD、CMD、KMD。定义一个JS 模块代码，最精简的格式是怎样。

(1) 你每天必须登录的网站（前端技术相关）是什么？

一加2--开箱

购买

我是一加的粉丝吗？谈不上。但是我确实很喜欢一加手机，可能是喜欢刘一虎的实诚劲儿，也可能是喜欢一加的扎实做工加上纯纯的安卓系统。一加一代的手机就买了俩，先买了个白色的babyskin，现在我妈在用。后来买了64g的砂岩黑，自己用了几个月，但是一代还是有一些问题的，首先就是不原生支持联通4g，每次刷系统都要打补丁，很烦，其次就是系统的问题，原生是oppo的colorOS,但是很明显不符合我的口味，于是开始了漫长的刷机之旅，在CM，miui，甚至iuni之间来回换，摇摆不定，朝秦暮楚，最后终于厌烦，把第二个一加手机出给了同学。

出手之后，就换成了魅族的MX4。用了几个月的MX4，明白了拥有的好你总是不知道有多好，直到失去了才知道珍惜。MX4的发热，卡顿真的是把我搞得不胜其烦。

这时时隔461天，一加开了发布会，发布了一加2手机，我看了发布会，感觉硬件上其实没有什么很惊艳的地方，反而是之前就一直惦记的H2OS实在是喜欢不已。转眼等到8月4号，一加2早上10点开售，我坐在电脑前，准时抢购，准时没抢到。。。

没抢到虽然很不心甘，但是本来就没打算买，抢购只是凑热闹而已。但是也在微博上吐槽了一下一加的抢购。这时，出事情了：一个同学说他有邀请码，问我要不要买。我正被没抢到而恼怒不已，听到这个当然要买！

于是，邀请码，下单，付款，一气呵成。8月6号，星期四，我买了一加手机2。买完我冷静下来想：我是不是太冲动了。。。。

等待

买完了就隔三小时看一下有没有发货，然后物流到哪了，真是牵肠挂肚。8月6号下单，结果到8月7号才出库。虽然也不慢，但是对于心急如焚的我来说，还是太慢了。然后发觉一个很折磨人的问题，8月7号是星期五，周末不送货，所以两天之后就能拿到的手机，我等了足足四天。周末的心情很矛盾，一方面希望周末慢慢的，享受不用工作的闲暇时光，另一方面又期盼着周一，希望早点拿到手机。

工作之后，第一次如此渴望着周一！！

开箱

周一，8月10号。

早上九点多到公司，就到放快递的地方，一遍翻，希望看到一加的身影。可惜，没看到，应该是还没有送到，安慰自己说一定就在路上，马上就到。

果然，过了没半个小时，可爱的顺丰就送来了包裹，马上签收，然后暴力撕开外面的塑料袋，撕开类似亚马逊kindle封口的一次性外箱封口，终于看到了一加2的盒子，红色的！！！依然是那一抹亮丽的红色。

相比一加一代，二代的盒子要质朴简约很多，这也是进步吧，一代的保重总是很隆重，比如锤子，比如bong。以后就越来越知道怎样才是成本和品牌的最佳平衡点。

撕掉外面的塑料膜，打开盒子，一加2静静地躺在盒子里。

拿出手机，手机下是说明书之类的纸质说明，再下面就是Type C的数据线和充电接头。其实Type C挺让我苦恼的，以后就要随身携带这根数据线了吗？很不方便啊，再买一根的话，又是开销啊，真是苦恼。

来一个主要成员的全家福。

现在再来看主角手机，整体的感觉很扎实，拿掉外面的保护膜，一加官方还在屏幕上贴了一张保护膜，省去了自己贴膜，赞！！背面是砂岩黑的材质，我其实最喜欢黑杏的后盖，很有气质。但是想到是可以换盖的，所以先体验砂岩黑，等厌烦了之后再买个黑杏的后盖，完美计划！

感受完后盖，就赶紧开机看看，一开机就是一加的标志。

然后就是肝儿颤的H2OS的开机动画，真的很喜欢。

开机之后，H2OS果然不负我的期望，整体特别素雅，基本没有内置应用，及其简洁。急忙录入指纹，指纹解锁也很快，感觉整个系统很棒。

总结

初步上手，感觉还是很棒的。现在最担心的就是三点了：续航，发热，和拍照。这些都是要在以后使用中在感受体会的。

现在要做的就是赶紧找一家联通营业厅，把我的大卡换成小卡，就可以用起来一加2了，哈哈哈哈哈！！！

我的数码生活（一）

序

对于懒惰的我，坚持写博客真的似乎是不可能的事。总是想写些东西，记录自己百无聊赖的生活，这生活如此乏味，我自己不记录恐怕永远都会消失在时间里，等到在我笨笨的脑袋慢慢地淡忘往事，它们就真的凭空消失了，一点痕迹都没有留下，无影无踪，无处找寻，想到这里，背脊一阵发凉，于是赶紧勤劳起来，记录下还没消失的生活。

既然要写，总要有主题，能唤起我书写欲望的，你们也看到了，就如题目写的：我的数码生活。

起始

我何时开始对数码感兴趣，要追溯到高中时代。高二前，当我还没遇到改改之前，根本不知数码为何物。那是2008年，iPhone也才在前一年发布，诺基亚还是曾经的巨头，嘲笑着苹果的玩物手机。安卓还是一个带着全键盘的怪胎，黑莓也像诺基亚一样风头正劲。世界上还没有移动互联网这个词汇，用手机最多也是最潮的似乎就是上QQ，一切看起来都像是移动互联网的蛮荒时代。当然，我就像当时的诺基亚一样浑然不知，就像不懂NBA的女孩子不会知道挡拆、扣篮这些词汇的意义一样，我也不会知道苹果黑莓这些词汇除了水果之外的别的意义。

杭州的春天

杭州的春天来了。

经过了寒冷的难熬的寒冬，春天的到来更显得弥足珍贵，冬天阴冷暗淡的太阳不知被谁杀死了，而新上任的太阳是一位青春妙龄的少女，温柔曼妙，婀娜多姿，总是让人忍不住得要待在她的怀抱里，醉死在温柔乡。来自北方的我小时候读课文总是读到“春天来了，到处生机勃勃”，可现实里却是寒冷的冬天慢吞吞得赖着不走，夏天就气势汹汹的来了。春天总是匆匆，仿佛夹在中间受气的小姑娘。人们还没来得及感受到处的生机勃勃，就被粗鲁的夏天轰出了人间，让人猝不及防。但杭州的春天不一样，她像一幅画卷，在慢慢地展开，不急不缓，让人领略到每一处的风采，享受到每一处的滋味，沉浸入春天的温暖幸福的氛围里。

第一次感受到杭州的春天是在一个早晨，住在三楼的我还没睡醒，就隐约感到拉住的窗帘都挡不住的灿烂的阳光，睁开眼看，窗帘已被照耀成一股温柔的暖黄色，从拉紧的窗帘的缝隙还偷偷钻进一束阳光，照到我的被子上，成一块光斑，我不禁用手去触碰，冬日沁凉的被子的感觉已消失的无影无踪，入手一片温暖，顿时暖进了心底里，暖得我睡意全消，于是伸懒腰，起床。

起来就迫不及待的拉开窗帘，嚇！一片温柔的金黄色，阳光照耀在每一处楼房，由远及近，从高到低。我的屋子里立刻便被灌满了温暖的感觉，就这么突然间，冬日的阴沉消失了，春天就这么duang的一下来了！路上的行人似乎也比往日多了，大家都嘴角上扬，看起来都和我一样沉浸在一片春意中。而马上我的注意力就被楼下一片白色吸引，楼下两座单云楼之间种着两颗树，我学识浅薄，不知是什么树，但现在满满当当都开出了白的的花，从我三楼的视角俯视好不壮观，但花刚刚开，还透着一股娇嫩。微风吹来，满树的花朵随风微微摇晃。我立刻打开了窗户，微风拂面，是春姑娘温柔的抚摸吧，一股花香也随风飘进了房间，是春姑娘清新的体香吧。

突然想起一句诗“忽如一夜春风来，千树万树梨花开”，这是白的梨花形容瑞雪，但现在的情景却是“就是一夜春风来，一树两数梨花开”，现在，我确信，是的！春天来了！是的！夏天还远！我迫不及待的出门了，去到这杭州美丽的春天里了。

2015-03-20

记录博客发布流程

距离上一次写博客又过去了差不多两个月，有一个笑话是程序员花了两天的时间各种搞，终于搭起了自己的个人博客，然后再也没更新过。。。我现在就变成了这个笑话。因为更新不频繁，所以每次更新都要百度一下hexo的指令，所以索性在这里记录一下，下一次直接看这篇东西就好了。

准备

首先hexo是要求node环境的，我的电脑是已经安装了node，所以直接跳过这一步。GitHub方面我也是准备好的，所以万事俱备，只欠“hexo”。

Hexo安装与配置

npm install hexo -g #-g表示全局安装, npm默认为当前项目安装
Hexo使用命令：
hexo init <folder> #执行init命令初始化hexo到你指定的目录
hexo generate #自动根据当前目录下文件,生成静态网页
hexo server #运行本地服务
浏览器输入http://localhost:4000就可以看到效果。

添加新文章

hexo new "postName" #新建博文,其中postName是博文题目
博文会自动生成在博客目录下source/_posts\postName.md
#文件自动生成格式:
title: "It Starts with iGaze: Visual Attention Driven Networkingwith Smart Glasses" #博文题目
date: 2014-11-21 11:25:38 #生成时间
tags: Paper #标签, 多个标签使用格式[Paper1, Paper2, Paper3,...]
---
如果不想博文在首页全部显示, 并能出现阅读全文按钮效果, 需要在你想在首页显示的部分下添加<!--more-->

主题更改

Hexo提供了官网的主题, 初始化hexo时也会自动生成一个主题, Hexo还支持个性定制主题, 可以根据自己的喜好对主题进行修改, 更多主题可以在官网中找到

个性化博客的设置

在博客的根目录下对喜爱的主题进行主题进行克隆
git clone git@github.com:yunlzheng/hexo-themes.git themes/writing

#在./_config.yml，修改主题为writing
theme: writing

#查看本地效果
hexo g #hexo generate简写
hexo s #hexo server简写

部署到Github

以上内容都是在本地进行查看, 现在将博客部署到github上
打开./_config.yml
在配置文件中设置号部署设置后,
hexo deploy #进行部署

总结

经过一番梳理，大概的流程是这样的：

  • 打开Ulysses,尽情挥洒文采（呵呵呵）
  • 保存到相应文件夹
  • hexo g
  • hexo s 本地预览
  • hexo d

2014-12-15日，记

今天是周一。

美剧

周末两天在租的房子里待了两天没出门，因为身体不舒服，要静养。看了一个美剧，叫《复仇》，感觉没有想象中有意思，我还是喜欢像《国土安全》这一类型的美剧，情节一以贯之，节奏紧张，应该是上周末，看了第四季《国土安全》的前九集，真的是一口气看完，实在是过瘾，但再看别的就有点食之无味的感觉。
总之现在还在找一个新的美剧，让我沉浸其中，不能自拔。

关于美剧，我最近比较喜欢的几部有疑犯追踪，国土安全，权利的游戏，绝命毒师，纸牌屋。现在的新剧感觉都一般。

阅读

周六的时候还把《碧血剑》看完了，我的宏大的“金庸作品重读计划”已经完成了《鹿鼎记》和《碧血剑》，下一部准备看《侠客行》。很明显，《鹿鼎记》比《碧血剑》好看得多，感觉后者还是很稚嫩的，还在老派武侠的感觉里没走出来，前者则是金庸后期非常成熟的作品。

游戏

周日的时候在PS Store上买了数字般的gta5，于是噩梦开始了，下载44g的东西，就靠我租的房子那里孱弱的网络，下了一晚上，终于开始安装，以为马上可以玩了，又开始了漫长的安装过程，一晚上安装了10%。周一早上开着ps4主机就来工作了，希望晚上回去会有惊喜。

工作

上周的和bong在一起的分享页面在手机上css3动画失效，这源于我的一个低级的失误，keyframe竟然没有加各个浏览器的适配，导致在电脑上是可以的，但在手机上是失效的，于是加上了-webkit-，-moz-,-o-。这种低级错误希望以后不要再犯了。

然后研究了一下在网页上实现ios7的毛玻璃效果，基本上是以下css代码：

-webkit-filter:blur(10px);
filter:url(#blur-effect);
opacity:.35;

但是奇妙的是可以实时渲染，可以看这个网址ios毛玻璃。
